A continuaci√≥n se muestra c√≥mo hacer un bucle de entrenamiento para personalizarlo al m√°ximo. Se asume que se ha heco el [procesamiento de datos](./1.%20Processing_data.md)
```python
from datasets import load_dataset
from transformers import AutoTokenizer, DataCollatorWithPadding

raw_datasets = load_dataset("glue", "mrpc")
checkpoint = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(checkpoint)


def tokenize_function(example):
    return tokenizer(example["sentence1"], example["sentence2"], truncation=True)


tokenized_datasets = raw_datasets.map(tokenize_function, batched=True)
data_collator = DataCollatorWithPadding(tokenizer=tokenizer)
```


## Prepare for training
Debemos definir un par de objetos como los dataloaders para iterar sobre los batches. Sin embargo, antes debemos modificar un poco tokenized_datasets para encargarnos de las cosas que *Trainer* hac√≠a. Espec√≠ficamente:
1. Eliminar columnas de valores que el modelo no espera (sentence1 y sentence2).
2. Renombrar la columna label a labels.
3. Cambiar el formato de datasets para que devuelvan Pytorch tensors en lugar de listas.
```python
tokenized_datasets = tokenized_datasets.remove_columns(["sentence1", "sentence2", "idx"])
tokenized_datasets = tokenized_datasets.rename_column("label", "labels")
tokenized_datasets.set_format("torch")
tokenized_datasets["train"].column_names
```

Definir dataloaders:
```python
from torch.utils.data import DataLoader

train_dataloader = DataLoader(
    tokenized_datasets["train"], shuffle=True, batch_size=8, collate_fn=data_collator
)
eval_dataloader = DataLoader(
    tokenized_datasets["validation"], batch_size=8, collate_fn=data_collator
)
```
Para revisar que no haya fallos podemos inspeccionar un batch de la siguiente forma:

Las *shapes* probablemente cambiar√°n poque usamos shuffle=True y estamos haciendo padding al max length de cada batch

Ahora en lo referente al modelo:
```python
from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
```

Para asegurarnos de que todo funciona correctamente probamos el batch:
```python
outputs = model(**batch)
print(outputs.loss, outputs.logits.shape)
```
    tensor(0.5441, grad_fn=<NllLossBackward>) torch.Size([8, 2])

> All ü§ó Transformers models will return the loss when labels are provided, and we also get the logits (two for each input in our batch, so a tensor of size 8 x 2).

Finalmente, para el bucle de entrenamiento solo nos faltan un optimizador y un learning rate scheduler. En este caso vamos a usar AdamW
```python
from transformers import AdamW

optimizer = AdamW(model.parameters(), lr=5e-5)
```

Finalmente, el learning rate scheduler usado por defecto es es *llinear decay* desde un valor m√°ximo de 5e-5 a 0. Para definirlo adecuadamente, debemos saber el n√∫mero de training steps que haremos:
```
training_steps = n_epochs ¬∑ n_batches
```
Donde el n√∫mero de batches es la longitud de nuestro training dataloader.  
El Trainer usa 3 √©pocas por defecto as√≠ que seguiremos ese ejemplo:
```python
from transformers import get_scheduler

num_epochs = 3
num_training_steps = num_epochs * len(train_dataloader)
lr_scheduler = get_scheduler(
    "linear",
    optimizer=optimizer,
    num_warmup_steps=0,
    num_training_steps=num_training_steps,
)
print(num_training_steps)
```

## Training loop
Si tenemos acceso a una GPU **debemos usarla** 
```python
import torch

device = torch.device("cuda") if torch.cuda.is_available() else torch.device("cpu")
model.to(device)
device
```
    device(type='cuda')

Listos para el entrenamiento, a√±adiendo una barra de progreso de tqdm library:
```python
from tqdm.auto import tqdm

progress_bar = tqdm(range(num_training_steps))

model.train()
for epoch in range(num_epochs):
    for batch in train_dataloader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)
        loss = outputs.loss
        loss.backward()

        optimizer.step()
        lr_scheduler.step()
        optimizer.zero_grad()   # No olvidar resetear los gradientes una vez el step del optimizer est√° hecho
        progress_bar.update(1)
```
El √∫nico problema es que esto no reporta nada sobre el entrenamiento (sin evaluaci√≥n)


## Evaluation loop
Usando la librer√≠a ü§ó *Evaluate*. Ya emos visto metric.compute() pero las m√©tricas pueden acumular batches usando add_batch(). En este caso, tras acumular todos los batches usaremos metric_compute.
```python
import evaluate

metric = evaluate.load("glue", "mrpc")
model.eval()
for batch in eval_dataloader:
    batch = {k: v.to(device) for k, v in batch.items()}
    with torch.no_grad():
        outputs = model(**batch)

    logits = outputs.logits
    predictions = torch.argmax(logits, dim=-1)
    metric.add_batch(predictions=predictions, references=batch["labels"])

metric.compute()
```
    {'accuracy': 0.8431372549019608, 'f1': 0.8907849829351535}


## Supercharge your training loop with ü§ó Accelerate
Usando la librer√≠a *accelerate* podemos activar entrenamiento distribuido sobre varias GPUs o TPUs. El c√≥digo que hemos hecho hasta este punto tiene esta pinta:
```python
from transformers import AdamW, AutoModelForSequenceClassification, get_scheduler

model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
optimizer = AdamW(model.parameters(), lr=3e-5)

device = torch.device("cuda") if torch.cuda.is_available() else torch.device("cpu")
model.to(device)

num_epochs = 3
num_training_steps = num_epochs * len(train_dataloader)
lr_scheduler = get_scheduler(
    "linear",
    optimizer=optimizer,
    num_warmup_steps=0,
    num_training_steps=num_training_steps,
)

progress_bar = tqdm(range(num_training_steps))

model.train()
for epoch in range(num_epochs):
    for batch in train_dataloader:
        batch = {k: v.to(device) for k, v in batch.items()}
        outputs = model(**batch)
        loss = outputs.loss
        loss.backward()

        optimizer.step()
        lr_scheduler.step()
        optimizer.zero_grad()
        progress_bar.update(1)
```

Este ser√≠a el c√≥digo a secas con accelerate:
```python
from accelerate import Accelerator
from transformers import AdamW, AutoModelForSequenceClassification, get_scheduler

accelerator = Accelerator()

model = AutoModelForSequenceClassification.from_pretrained(checkpoint, num_labels=2)
optimizer = AdamW(model.parameters(), lr=3e-5)

train_dl, eval_dl, model, optimizer = accelerator.prepare(
    train_dataloader, eval_dataloader, model, optimizer
)

num_epochs = 3
num_training_steps = num_epochs * len(train_dl)
lr_scheduler = get_scheduler(
    "linear",
    optimizer=optimizer,
    num_warmup_steps=0,
    num_training_steps=num_training_steps,
)

progress_bar = tqdm(range(num_training_steps))

model.train()
for epoch in range(num_epochs):
    for batch in train_dl:
        outputs = model(**batch)
        loss = outputs.loss
        accelerator.backward(loss)

        optimizer.step()
        lr_scheduler.step()
        optimizer.zero_grad()
        progress_bar.update(1)
```


## Last notes:
Putting this in a train.py script will make that script runnable on any kind of distributed setup. To try it out in your distributed setup, run the command:

    accelerate config

which will prompt you to answer a few questions and dump your answers in a configuration file used by this command:

    accelerate launch train.py

which will launch the distributed training.

If you want to try this in a Notebook (for instance, to test it with TPUs on Colab), just paste the code in a training_function() and run a last cell with:
```python
from accelerate import notebook_launcher

notebook_launcher(training_function)
```
You can find more examples in the ü§ó Accelerate repo.
